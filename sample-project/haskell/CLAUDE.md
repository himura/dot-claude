# Haskell Conventions

## Language Extensions & Standards

### Record Access

Use `OverloadedRecordDot` with `NoFieldSelectors` for record field access:

```haskell
{-# LANGUAGE NoFieldSelectors #-}

data User = User
    { name :: Text
    , email :: Text
    }

-- Access with dot notation (like JavaScript/TypeScript)
getUserEmail :: User -> Text
getUserEmail user = user.email

-- Nested access
data Config = Config { database :: DatabaseConfig }
data DatabaseConfig = DatabaseConfig { host :: Text }

getHost :: Config -> Text
getHost config = config.database.host
```

**Note**: Field selector functions (e.g., `name :: User -> Text`) are NOT generated due to `NoFieldSelectors`.

### Deriving Strategies

Always explicitly specify deriving strategy:

```haskell
data Tag = Tag
    { name :: Text
    , category :: TagCategory
    }
    deriving stock (Generic, Show, Eq)
    deriving anyclass (Exception, NFData)

-- Standalone deriving when needed
deriving stock instance Eq (TagBodyT Identity)
deriving stock instance Show (TagBodyT Identity)
```

Available strategies:
- `stock` - Built-in GHC deriving (Eq, Show, Generic, etc.)
- `anyclass` - Empty instance via DeriveAnyClass (Exception, NFData, etc.)
- `newtype` - Coercing through newtype wrapper
- `via` - DerivingVia with custom wrapper (see JSON Serialization)

### Qualified Imports

Use consistent aliases for standard libraries:

```haskell
-- ByteString
import Data.ByteString qualified as S
import Data.ByteString.Char8 qualified as S8

-- ByteString (Lazy)
import Data.ByteString.Lazy qualified as L
import Data.ByteString.Lazy.Char8 qualified as L8

-- Text
import Data.Text qualified as T
import Data.Text.Encoding qualified as T
import Data.Text.IO qualified as T

-- Text (Lazy)
import Data.Text.Lazy qualified as TL
import Data.Text.Lazy.Encoding qualified as TL

-- Collections
import Data.Map qualified as Map
import Data.Set qualified as Set
```

## Project Libraries & Patterns

### Custom Prelude Module

The project uses a custom `Prelude` module that provides:

- **Effectful effect system**: `IOE`, `Eff`, `:>` for effect constraints
- **Optics utilities**: `lens`, `view`, `^.`, `&`, `.~`, `%~` for lens operations
- **Text interpolation**: `st`, `lt` for Shakespeare quasi-quotes
- **Standard Prelude**: All standard functions and types from base Prelude

The custom `Prelude` is automatically imported in all modules - no explicit import needed.

### Effectful Effect System

Define custom effects with `Effect` kind and interpret with handler:

```haskell
{-# LANGUAGE TemplateHaskell #-}

-- Define effect
data TagRepository :: Effect where
    CreateTag :: Text -> TagOrigin -> TagRepository m Tag
    LookupTagById :: TagId -> TagRepository m (Maybe Tag)
    DeleteTag :: TagId -> TagRepository m ()

makeEffect ''TagRepository

-- Implement interpreter
runTagRepository :: (MySQLService :> es) => Eff (TagRepository : es) a -> Eff es a
runTagRepository = interpret $ \_ -> \case
    CreateTag name origin -> do
        runDBInTransaction $ createTag' name origin
    LookupTagById tagId -> do
        runDBInTransaction $ lookupByTagId tagId
    DeleteTag tagId -> do
        runDBInTransaction $ deleteTagByTagId tagId

-- Use effect with type constraint
processTag :: (TagRepository :> es) => TagId -> Eff es ()
processTag tagId = do
    tag <- lookupTagById tagId  -- Generated by makeEffect
    -- ...
```

**Type constraint pattern**: Use `:> es` for effect constraints, not `Member` or `Has`.

### Database.Beam ORM

Use Beam's Higher-Kinded Data Types (HKD) pattern with `EntityEnvelope`:

```haskell
{-# LANGUAGE NoFieldSelectors #-}

-- Define entity body with HKD
data TagBodyT f = TagBody
    { name :: Columnar f Text
    , status :: Columnar f TagStatus
    , category :: Columnar f TagCategory
    , description :: Columnar (Nullable f) Text  -- Nullable field
    , imageCount :: Columnar f Int32
    }
    deriving stock (Generic)
    deriving anyclass (Beamable)

-- Type aliases
type TagT = EntityEnvelope TagBodyT
type Tag = TagT Identity
type TagId = PrimaryKey TagT Identity

-- Query with Beam DSL
lookupByTagName :: Text -> MySQLM (Maybe Tag)
lookupByTagName name =
    runSelectReturningOne $
        select $ do
            tag <- all_ imageDb.tableTag
            guard_ (tag.body.name ==. val_ name)
            return tag

-- Access entity fields
getTagName :: Tag -> Text
getTagName tag = tag.body.name  -- EntityEnvelope has .body field
```

**Key patterns**:
- Use `Columnar f Type` for polymorphic fields
- Use `Nullable f` for optional fields
- Access fields via `entity.body.fieldName`
- Use `all_`, `guard_`, `val_` in Beam queries

### Text Interpolation (Shakespeare)

Use `[st|...|]` for strict Text and `[lt|...|]` for lazy Text:

```haskell
import Text.Shakespeare.Text (st, lt)  -- Re-exported from Base

-- Strict Text interpolation
errorMsg :: TagId -> Text
errorMsg tagId = [st|Tag not found: #{tagId}|]

-- Lazy Text interpolation
logMsg :: Text -> Int -> TL.Text
logMsg name count = [lt|Processing #{name}: #{count} items|]

-- With conversion to ByteString
responseBody :: Text -> ByteString
responseBody msg = T.encodeUtf8 [st|Error: #{msg}|]
```

**Note**: `#{}` syntax is used for interpolation, similar to Ruby/Elixir.

### Exception Handling

Use `throwIO` from `Base.Exception` (wraps annotated-exception):

```haskell
import Base  -- Provides throwIO, Exception

data TagRepositoryException
    = TagNotFound TagId
    | TagNameConflict Text TagId TagId
    deriving stock (Generic, Show, Eq)
    deriving anyclass (Exception)

-- Throw exception with CallStack annotation
lookupTag :: TagId -> MySQLM Tag
lookupTag tagId = do
    lookupByTagId tagId >>= \case
        Nothing -> throwIO $ TagNotFound tagId
        Just tag -> return tag
```

**Never** use `throw` or `error` directly. Always use `throwIO` for automatic CallStack tracking.

### JSON Serialization

Use `DerivingVia` with `Default` wrapper for automatic JSON instances:

```haskell
import Data.Aeson.DerivingVia (Default)

data User = User
    { userId :: Int
    , userName :: Text
    , userEmail :: Text
    }
    deriving stock (Generic, Show)
    deriving (FromJSON, ToJSON) via Default User

-- Generates JSON with snake_case field names:
-- {"user_id": 1, "user_name": "Alice", "user_email": "alice@example.com"}
```

**Field naming convention**:
- Haskell fields use `camelCase` with type prefix (e.g., `userName`)
- JSON fields use `snake_case` with prefix stripped (e.g., `user_name`)
- Conversion is automatic via `projectDefaultOptions`
